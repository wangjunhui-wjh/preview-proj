# Qoder Work Skill 使用说明

已生成的 skill：

```text
qoder-work-skills/eia-precheck-assistant/
```

压缩包：

```text
outputs/eia-precheck-assistant-qoder-work-skill.zip
```

## Qoder Work 本地使用

将整个目录复制到：

```text
~/.qoderwork/skills/eia-precheck-assistant/
```

目录结构应保持：

```text
~/.qoderwork/skills/eia-precheck-assistant/SKILL.md
~/.qoderwork/skills/eia-precheck-assistant/references/
~/.qoderwork/skills/eia-precheck-assistant/templates/
```

## Qoder Cloud Agent 上传

上传：

```text
outputs/eia-precheck-assistant-qoder-work-skill.zip
```

压缩包内包含：

```text
eia-precheck-assistant/SKILL.md
eia-precheck-assistant/references/
eia-precheck-assistant/templates/
```

## 建议触发方式

在 Qoder Work 中输入类似：

```text
使用 eia-precheck-assistant skill，读取这些项目资料，按照 PREP-INGEST -> HB-PT-000 -> HB-PT-011 的环评前期研判流程执行，必要时联网搜索真实政策依据，并输出每个节点结果和综合报告。
```

或：

```text
使用环评前期研判 skill，仅运行 HB-PT-002 和 HB-PT-003，判断行业类别、环评类别、审批路径和产业政策风险。
```

## 注意

- 这个 skill 不包含当前 FastAPI/Hermes/LangGraph 服务代码，只沉淀工作流、节点提示词、证据规则和输出模板。
- 如需产品化 Web 页面、暂停恢复、历史记录、知识库审核等能力，仍使用当前仓库的 Docker Compose 系统。
- 依据必须来自上传资料、可读取文件、OCR/视觉结果、已验证本地知识库或真实访问的 URL。
