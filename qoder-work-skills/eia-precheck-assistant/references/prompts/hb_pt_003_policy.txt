【用户提示词 - HB-PT-003 产业政策符合性及政策有效性识别】

请调用 HB-PT-001 项目概况提取结果，并结合知识库中的产业政策文件，完成产业政策符合性分析，同时识别所引用政策文件的有效性。

【输入变量】
项目概况：
{project_profile}

知识库依据片段：
{evidence_context}

【执行规则】
1. 产业政策符合性必须基于 {project_profile} 中的行业、产品、产能、工艺、装备、原辅材料等事实，以及 {evidence_context} 中可核验的政策依据。
2. 政策文件有效性识别应检查文件名称、文号、发布日期、实施日期、有效期、废止说明、替代文件说明、知识库中是否存在更新文件。
3. 如知识库未检索到更新文件或废止依据，不得自行编造替代文件或废止公告，应写“有效性待核实，建议人工确认”。
4. 如 {evidence_context} 为空、缺少适用产业政策或条款不能支撑结论，符合性判断必须写“资料不足，建议人工核实”。
5. 不得编造政策文件名称、文号、条款号、发布日期、废止状态或替代关系。

【输出结构】
一、政策文件有效性识别表
请按“文件名称 / 文号 / 发布日期 / 实施日期 / 效力状态 / 识别依据 / 是否建议用于本次研判”输出。
效力状态只能选择：现行有效 / 疑似废止 / 已废止 / 已被替代 / 部分条款失效 / 有效性待核实。

二、产业政策符合性总判断
请在“鼓励类 / 限制类 / 淘汰类 / 允许类 / 待核实”中选择，并说明依据。

三、项目与政策条款对照
请按“项目事实 / 适用政策条款 / 符合性判断 / 风险等级”输出。

四、新旧文件替代关系
如存在新旧文件替代关系，请列出旧文件、新文件、替代依据、对本项目研判影响；如依据不足，输出“资料不足，建议人工核实”。

五、风险点、需补充资料、人工复核要点
请明确风险事项、风险等级、影响模块和建议补充资料。

【固定免责声明】
输出末尾必须追加：以上为AI辅助初步研判，最终结论需由环评工程师人工复核确认。
