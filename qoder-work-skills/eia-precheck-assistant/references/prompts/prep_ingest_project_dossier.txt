【用户提示词 - PREP-INGEST 项目资料读取与项目档案构建】

请基于用户提交的原始项目资料，完成“项目资料读取、OCR/视觉识别、事实归纳和来源索引”，形成后续 HB-PT 节点可复用的项目档案。

【输入变量】
用户粘贴的原始文本：
{project_text}

可选依据片段：
{evidence_context}

【执行边界】
1. 这是资料读取与项目档案构建节点，不做环评类别、产业政策、三线一单、两高或化工合规结论。
2. 项目事实必须来自用户粘贴文本、上传文件、PDF 文本层、扫描页 OCR/视觉识别或图片识别结果。
3. 不得为了补充项目事实而使用 web_search。外部网页不得替代项目原始资料。
4. PDF 先判断是否有文本层；有文本层则优先抽取文本，扫描页或图片页再使用 OCR/视觉识别。不得把整份 PDF 无差别当图片处理。
5. 对图片、图纸、扫描件、表格、章/签字页、截图等资料，应提取与项目研判有关的信息，并注明识别不确定性。
6. 无法确认的信息统一写“未提供”或“资料不足，建议人工核实”，不得根据行业经验补全。
7. 每个关键事实尽量记录来源：文件名、页码/图片名、表格名、可见文字或用户粘贴文本片段。

【输出要求】
一、资料包读取概况
请列出：资料来源类型、文件数量、已读取文件、疑似扫描件/图片页、OCR/视觉识别情况、无法读取或需人工复核的资料。

二、项目档案摘要
请用表格输出以下字段：字段 / 提取值 / 来源 / 可信度 / 备注。
至少包括：
- 项目名称
- 建设单位
- 建设地点
- 所在园区或片区
- 建设性质
- 行业或产品方向
- 主要产品及产能
- 主要建设内容
- 主要生产工艺
- 主要原辅材料
- 主要设备
- 用地面积或用地性质
- 总投资或环保投资
- 废气、废水、固废、噪声、环境风险相关信息
- 能耗、水耗、排污去向
- 周边敏感目标、生态红线、长江岸线等空间信息
- 资料中明确出现的政策、规划、批复、园区文件名称

三、资料矛盾与不确定项
列出资料内部不一致、表述模糊、扫描识别不确定、缺页、图像模糊、无法定位来源的事项。

四、后续研判可用输入
请整理为后续节点可直接使用的项目事实清单。只写已确认或带来源的不确定事实，不写推测。

五、需补充资料清单
请按“资料名称 / 用途 / 影响后续模块 / 优先级 / 建议提供方”输出。

【结构化 JSON】
请额外输出一个 JSON 代码块或 BEGIN_NODE_RESULT_JSON/END_NODE_RESULT_JSON，字段建议包括：
{
  "node": "PREP-INGEST",
  "project_dossier": {
    "basic_info": {},
    "construction_content": {},
    "process_and_materials": {},
    "pollution_and_controls": {},
    "spatial_info": {},
    "source_index": []
  },
  "uncertainties": [],
  "missing_materials": []
}

【固定免责声明】
输出末尾必须追加：以上为AI辅助资料读取和项目档案构建结果，最终项目事实需由环评工程师结合原始资料人工复核确认。
