【辅助提示词 - WEB-SEARCH 联网检索与候选依据发现】

请围绕用户给定的检索问题，使用 web_search/web_extract 检索官方政策、官方答复、政策解读、规划环评、审批原则或同类项目公开资料，形成可审核的候选依据清单。

【输入变量】
检索问题：
{query}

项目概况：
{project_profile}

已完成模块输出：
{module_outputs}

知识库依据片段：
{evidence_context}

【执行规则】
1. 由你根据检索问题和项目概况自主组织关键词，不要依赖前端限定词；优先检索生态环境部、国家发改委、工信部、自然资源部、地方生态环境局等官方来源。
2. 检索结果必须包含真实 URL、标题、来源机关或站点、摘要、与本项目或当前研判问题的相关性。
3. 如发现 PDF、附件或政策原文页面，优先读取可访问正文；无法抽取正文时记录候选 URL，并说明“原文抽取失败，建议人工核实”。
4. 不得编造政策名称、条款、官方答复或 URL；不确定时明确写“资料不足，建议人工核实”。
5. 输出的候选依据只是候选库资料，必须经人工审核后才可作为正式知识库依据。

【输出结构】
一、检索结论摘要
说明本次检索是否找到可用官方依据、主要依据类型、仍需人工核实的问题。

二、候选依据清单
请按“标题 / 来源 / URL / 摘要 / 相关性 / 建议用途 / 是否需人工审核”输出，建议 5-10 条，宁缺毋滥。

三、推荐检索词和未解决问题
列出你实际使用或建议继续使用的检索词，以及仍未找到充分依据的问题。
