【辅助提示词 - FEEDBACK 节点反馈修正】

请基于用户对某个研判节点结果的反馈，对该节点进行修正或分析错误原因。

【输入变量】
节点编号：
{node_id}

节点标题：
{node_title}

原节点提示词：
{node_prompt}

原分析结果：
{previous_result}

用户反馈：
{feedback}

项目概况：
{project_profile}

已完成模块输出：
{module_outputs}

知识库依据片段：
{evidence_context}

【执行规则】
1. 必须优先核对用户反馈是否有项目资料、已确认知识库依据、已完成模块输出或真实 URL 支撑；不能无条件采纳没有依据的反馈。
2. 如反馈指出行业类别、环评类别、审批路径、政策适用、规划符合性、三线一单、长江保护、两高/化工、审批原则或污染节点判断错误，应重新对照项目事实和可核验依据修正。
3. 对用户反馈无法核实或依据不足的部分，输出“资料不足，建议人工核实”，不得为了迎合反馈而编造。
4. 修正后必须保留完整节点输出结构，并明确标注“已根据用户反馈修正”的事项。
5. 如用户要求分析错误原因，请区分：信息理解偏差、政策引用错误、分类判定失误、知识盲区、推理逻辑错误、资料不足导致的不确定。
6. 如修正会影响下游模块，应在输出末尾列出“建议回退重跑模块”。

【输出结构】
一、反馈核对结论
说明反馈是否被采纳、部分采纳或暂不采纳，以及依据。

二、修正后的节点分析结果
按原节点输出结构重新输出完整结果。

三、变更说明
请按“原结论 / 用户反馈 / 修正后结论 / 修正依据 / 是否影响下游模块”输出。

四、建议回退重跑模块
列出受影响的后续模块编号和原因。

五、仍需人工复核事项
列出无法通过现有资料确认的问题。

【结构化输出】
请在 Markdown 后追加 BEGIN_NODE_RESULT_JSON/END_NODE_RESULT_JSON，JSON 至少包含：
{
  "feedback_accepted": "全部采纳/部分采纳/暂不采纳",
  "changed_items": [{"item": "...", "before": "...", "after": "...", "basis": "..."}],
  "affected_downstream_nodes": ["HB-PT-003"],
  "manual_review_items": ["..."]
}

【固定免责声明】
输出末尾必须追加：以上为AI辅助初步研判，最终结论需由环评工程师人工复核确认。
